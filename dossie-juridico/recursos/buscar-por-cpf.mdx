# 🔎 Buscar por CPF (Parte)

Este endpoint permite consultar todos os processos em que um determinado CPF esteja envolvido, como parte **ativa**, **passiva** ou **outros**.

Você pode aplicar filtros por grau, tribunal, classe, assunto, valor da causa, status, ramo do direito e mais.

---

## 🔗 Endpoint

```
POST https://api.predictus.inf.br/predictus-api/processos/judiciais/buscarPorCPFParte
```

## 📚 Campos importantes

- `cpf` (string) — único campo obrigatório
- `polo` — `ATIVO` ou `PASSIVO`
- `grausProcesso` — lista de graus processuais. Valores possíveis:
  - `1` → 1º Grau (ex: vara cível, vara do trabalho)
  - `2` → 2º Grau (ex: tribunais de justiça ou regionais)
  - `3` → Tribunais Superiores (STJ, TST, STM, TSE)
  - `4` → Supremo Tribunal Federal (STF)
- `palavrasChave` — lista de termos buscados nos **assuntos** e **movimentações** do processo.  
  > Os termos são combinados com lógica **OR**. Se qualquer um for encontrado, o processo será retornado.
- `tribunais` — veja [lista completa de tribunais](../referencias/tribunais)
- `valorCausa`, `dataDistribuicao` — permitem filtragem por faixa numérica ou temporal
- `classesProcessuais` — veja [lista de classes](../referencias/classes-processuais)
- `assuntosCNJ` — veja [lista de assuntos CNJ](../referencias/assuntos)
- `statusProcessuais` — veja [status possíveis](../referencias/status-processuais)
- `ramosDoDireito` — veja [ramos do direito](../referencias/ramos-direito)

---

## 📌 Como funcionam os filtros

A API aplica os filtros com **lógica combinada (AND entre filtros diferentes, OR dentro de listas)**:

| Situação                         | Comportamento                                             |
|----------------------------------|------------------------------------------------------------|
| Vários filtros usados            | Todos devem ser satisfeitos (lógica **AND**)              |
| Lista de valores (ex: tribunais) | Basta bater **um dos valores** da lista (lógica **OR**)   |
| Campos únicos (ex: CPF)          | Precisa ser exatamente igual                              |
| Faixas (valor ou data)           | O valor do processo deve estar **dentro do intervalo**    |

### 🧠 Exemplos práticos

**Exemplo 1 — Consulta com `ramosDoDireito` e `classesProcessuais`:**

```json
{
  "cpf": "08108732905",
  "ramosDoDireito": ["DIREITO DO TRABALHO", "DIREITO CIVIL"],
  "classesProcessuais": ["CUMPRIMENTO DE SENTENCA"]
}
```

🔍 Resultado:
- Todos os processos **do CPF informado**
- **E** que tenham **classe = CUMPRIMENTO DE SENTENCA**
- **E** cujo **ramo do direito** seja **TRABALHO ou CIVIL**

---

**Exemplo 2 — Consulta com `valorCausa` e `statusProcessuais`:**

```json
{
  "cpf": "08108732905",
  "valorCausa": { "maiorQue": 5000 },
  "statusProcessuais": ["EM TRAMITACAO", "ARQUIVAMENTO DEFINITIVO"]
}
```

🔍 Resultado:
- Todos os processos do CPF
- Com valor da causa maior que 5.000
- E cujo status seja **um dos listados**

---

> **Resumo:**
> - **Entre filtros diferentes → E (AND)**  
> - **Dentro de listas → OU (OR)**

---

## 🧾 Retorno

Retorna uma lista de objetos do tipo [`processo`](./estrutura-dados), com dados estruturados e padronizados pela Predictus.
