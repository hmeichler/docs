# 🔎 Buscar por Número CNJ

Este endpoint recebe o identificador de um processo judicial e retorna um **resumo gerado por IA** com análise em linguagem acessível (`resumoRH`) e decomposição técnica estruturada (`juridico`).

> ⚠️ **Conteúdo gerado por IA.** Todo o texto desta resposta é produzido por modelos de linguagem e pode conter **erros, omissões e alucinações**. Não é parecer jurídico: revise contra os autos antes de qualquer decisão, e confira valores, datas e nomes na fonte primária. Leia os limites de uso na [introdução do Resume AI](../overview).

---

## 🔗 Endpoint

```
POST https://api.predictus.com.br/predictus-api/processos/judiciais/resumeai/buscarPorNumeroCNJ
```

---

## 🧾 Corpo da requisição

```json
{
  "numeroProcessoUnico": "10004588220255020441",
  "sistema": "PJE-TRT2",
  "tribunal": "TRT-2",
  "grauProcesso": 1,
  "dataDistribuicao": "2025-04-24"
}
```

---

## 📚 Campos

| Campo | Obrigatório | Tipo | Descrição |
|-------|-------------|------|-----------|
| `numeroProcessoUnico` | Sim | string | Número do processo no formato CNJ (20 dígitos numéricos, sem pontuação) |
| `tribunal` | Sim | string | Código do tribunal (ex: `TRT-2`, `TJSP`, `STJ`) — ver [lista de tribunais](../../referencias/tribunais) |
| `sistema` | Sim | string | Sistema de origem do processo (ex: `PJE-TRT2`, `ESAJ`, `PROJUDI`) |
| `dataDistribuicao` | Sim | string | Data de distribuição do processo no formato `YYYY-MM-DD` |
| `grauProcesso` | Recomendado | integer | Grau do processo: `1` (1º grau), `2` (2º grau), `3` (instância especial), `4` (instância extraordinária) |

---

## 📥 Exemplo de cURL

```bash
curl --location 'https://api.predictus.com.br/predictus-api/processos/judiciais/resumeai/buscarPorNumeroCNJ' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {accessToken}' \
--data '{
  "numeroProcessoUnico": "10004588220255020441",
  "sistema": "PJE-TRT2",
  "tribunal": "TRT-2",
  "grauProcesso": 1,
  "dataDistribuicao": "2025-04-24"
}'
```

## 🧾 Campos da resposta

### Raiz

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `resumoRH` | string | Resumo narrativo em linguagem acessível para leitores não jurídicos |
| `juridico` | object | Análise jurídica técnica estruturada |

### `juridico`

| Campo | Tipo | Descrição |
|-------|------|-----------|
| `decisoesAntecipadas` | string | Decisões liminares ou tutelas antecipadas. Retorna `"Não consta."` se não houver |
| `fatos` | string | Narrativa dos fatos relevantes do processo |
| `fundamentos` | string | Base legal aplicada: artigos de lei, súmulas, OJs e precedentes citados |
| `pedidos` | string | Todos os pedidos formulados com valores atribuídos à causa |
| `condenacoes` | string | O que foi deferido e indeferido, com valores arbitrados |
| `recursos` | string | Recursos interpostos pelas partes e estado atual |
| `linhaDoTempo` | array\<string\> | Cronologia dos eventos no formato `YYYY-MM-DD – TIPO: Descrição` |

---

## 🧾 Códigos de resposta

| Código | Significado |
|--------|-------------|
| `200` | Sucesso — resumo gerado com sucesso |
| `204` | Processo não encontrado ou sem documentos disponíveis para análise |
| `400` | Erro de validação no payload (campo obrigatório ausente ou inválido) |
| `401` | Token inválido ou expirado |
| `500` | Erro interno do servidor |

---

## 🛡️ Validações

- `numeroProcessoUnico`: não nulo, exatamente 20 dígitos numéricos no padrão CNJ
- `sistema`: não nulo
- `tribunal`: não nulo — ver [lista completa de tribunais](../../referencias/tribunais)
- `dataDistribuicao`: não nulo, formato `YYYY-MM-DD`, não pode ser data futura
- `grauProcesso`: quando informado, valor entre `1` e `4`

> Os quatro campos além do `numeroProcessoUnico` identificam a **instância exata** do processo — o mesmo número CNJ pode existir em mais de um grau ou sistema. Todos eles vêm prontos na resposta do [Dossiê Jurídico](../../dossie-juridico/overview): reaproveite os valores de lá em vez de montá-los à mão.

---

## 💡 Dica

> O campo `resumoRH` é ideal para exibir em dashboards, relatórios executivos e notificações para gestores. Já `juridico.linhaDoTempo` pode ser usado diretamente para montar uma visualização cronológica do processo em sistemas de gestão de passivos.
